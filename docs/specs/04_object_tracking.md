# 人体跟踪模块需求规格说明书 (SRD)

## 1. 目标 (Objective)
在相机端侧人体检测结果缺失（如网络丢包、端侧算力限制）时，利用本地解码后的视频帧执行人体检测，并集成 **ByteTrack** 跟踪算法，确保在复杂监控场景下（如遮挡、低置信度检测）实现稳定、连续的人体目标 ID 关联。

## 2. 系统接口 (Interfaces)

### 2.1 输入参数
| 参数名 | 数据类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| `decoded_frame` | `cv::Mat` | 是 | 当前时刻解码后的原始图像（用于特征提取或可视化调优） |
| `det_boxes` | `std::vector<Box>` | 是 | 本地检测器输出的原始框集合，每个 Box 需包含：`{x, y, w, h, score, label}` |

### 2.2 输出参数
| 参数名 | 数据类型 | 说明 |
| :--- | :--- | :--- |
| `tracks` | `std::vector<Track>` | 跟踪结果集合，每个 Track 包含：`{track_id, x, y, w, h, score, state}` |

## 3. 核心算法逻辑 (Core Logic)

### 3.1 ByteTrack 算法实现
* **双阶关联 (Two-stage Association)**：
    * **第一阶段**：将高置信度检测框与现有轨道（Tracks）通过 IoU 匹配。
    * **第二阶段**：将低置信度检测框（容易被误认为背景的遮挡目标）与第一阶段未匹配成功的轨道进行二次匹配，最大程度减少因遮挡导致的断连。
* **运动模型 (Motion Model)**：使用卡尔曼滤波（Kalman Filter）对人体运动进行状态估计（位置与速度）。
* **代价函数 (Cost Function)**：采用 IoU（交并比）或 DIoU 矩阵作为匈牙利算法的输入，计算全局最优匹配。

### 3.2 轨道状态机管理
每个目标必须处于以下状态之一：
* **Tentative (探测态)**：新检测出的目标，需连续出现 $N$ 帧才确认为有效跟踪。
* **Tracked (跟踪态)**：当前帧成功匹配的目标。
* **Lost (丢失态)**：当前帧匹配失败，但在最大保留时长（Max Age）内的目标。
* **Removed (删除态)**：超过保留时长或移出边界的目标，ID 将被释放。

## 4. 接口功能要求 (Functional Requirements)

### 4.1 Update 接口
* **功能**：处理观测值并执行完整闭环逻辑。
* **流程**：`卡尔曼预测` -> `双阶匹配` -> `状态更新` -> `卡尔曼修正`。
* **健壮性**：即使 `det_boxes` 为空，也必须执行卡尔曼预测以维持现有轨道状态。

### 4.2 Predict 接口
* **功能**：支持跳帧检测模式（Inference Skip）。
* **逻辑**：当上层跳过检测环节时，调用此接口利用卡尔曼滤波的先验速度预测目标在当前帧的虚拟位置。
* **限制**：需维护 `consecutive_predict_count` 计数，防止因长时间无观测值导致预测框发散。

## 5. 非功能性要求 (Non-functional Requirements)

* **配置化参数**：以下参数需支持配置文件读取：
    * `high_threshold`: 高置信度阈值。
    * `low_threshold`: 低置信度阈值。
    * `match_threshold`: IoU 匹配阈值。
    * `max_age`: 目标丢失后保留的最大帧数。
* **线程安全**：模块需支持在多线程解码流水线中稳定运行。

## 6. 验收标准 (Acceptance Criteria)
1.  **ID 稳定性**：在人体短时间被电线杆、绿植等遮挡（< 30帧）后恢复可见，ID 保持不变。
2.  **轨迹平滑度**：输出的目标框坐标应经过滤波处理，无肉眼可见的剧烈跳动。
3.  **性能达标**：在单路 1080P 视频、跟踪 15 人的情况下，`Update` 耗时需在 $100$ ms 以内。
4.  **接口完备性**：`Predict` 预测框在跳帧期间与后续检测到的真实框位置偏差在 $5\%$ 以内。
5.  **支持配置是否使用**： 需要支持通过配置是否使用跟踪模块。
