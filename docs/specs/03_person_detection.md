# 基于 RK3566 的人体检测模块

## 1. 目标
当相机侧无法提供人体检测结果时，系统应在本地对解码后的图像执行人体检测。

该模块运行于 **RK3566 + Android** 平台，基于瑞芯微提供的推理 API 实现板端模型加载与推理，并支持对 **YOLOv3、YOLOv5、YOLOv8** 模型输出结果进行后处理解析，提取人体目标框。

---

## 2. 输入

- `decoded_frame: cv::Mat`  
  解码后的当前视频帧图像
- `detector_config: DetectionConfig`  
  检测配置项，包括检测帧率、超时时间、丢帧策略、置信度阈值等

---

## 3. 输出

- `boxes: std::vector<BoundingBox>`  
  当前帧中检测到的所有 `person` 类目标框集合

---

## 4. 处理规则

- 按照配置的检测频率执行检测。  
  例如：当配置值为 `3` 时，表示每 `3` 帧检测一次，即检测帧序号为 `0、3、6、9 ...`。
- 检测结果仅保留类别为 `person` 的目标框。
- 当返回的 `boxes` 为空时，不进入后续 **Ref 帧比较** 和 **事件检测** 流程。
- 当相机侧检测结果可用时，相机检测结果需要基于视频帧的 **帧号** 或 **时间戳** 进行匹配与同步。
- 本模块部署平台为 **瑞芯微 RK3566 芯片**，操作系统为 **Android**。
- 图像预处理letterbox策略resize，然后除以255.进行归一化

---

## 5. 性能要求

- 检测超时时间应支持配置。
- 当处理链路出现积压时，允许根据配置的策略进行丢帧处理。
- 模型文件需提前通过瑞芯微提供的工具完成转换，并能够在 **RKNN Runtime** 上直接加载和运行。
- 模块应具备稳定运行能力，满足持续视频流场景下的检测需求。

---

## 6. 验收标准

- 当相机不支持输出人体检测结果时，本地人体检测流程能够稳定运行。
- 当检测到人体目标时，系统能够进入后续帧差分析流程，并与 `Ref` 帧进行相似性比较。
- 当未检测到人体目标时，系统跳过后续分析，继续处理下一帧。
- 系统能够记录检测结果日志，日志内容至少包括：
  - 帧号或时间戳
  - 检测到的人体数量
  - 每个目标框的位置及置信度
- 配置跳帧时，例如3，则每3帧检测一次，只在帧号模3为0时如0，3，6，9等帧进行检测
- RKNN的模型在配置文件中的**model_path**中指定，格式是**.rknn**,输入输出格式查询RKNN的文档

---

## 7. 说明

- 本模块仅负责本地人体检测，不负责事件检测本身。
- 本模块输出的目标框结果将作为后续 **Ref 帧过滤** 和 **事件分析** 的输入依据。
