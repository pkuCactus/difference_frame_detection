# 视频流帧差分析系统

## 1. 目标

开发一套基于 **SDC 相机 G60 + RK3566** 平台的单路视频帧差分析系统。

系统通过 RTSP 从相机拉取视频流，优先使用相机侧返回的人体检测结果；若相机不支持，则回退到 RK3566 本地执行人体目标检测（可选启用跟踪模块）。

仅当检测到人体目标，且当前帧与保存的参考帧（Ref）存在显著差异时，才将当前帧或当前帧之后的一段短视频送入事件分析模块。

---

## 2. 范围

### 本期实现

- 单路相机 RTSP 拉流与解码
- 相机侧人体检测结果获取（ONVIF / REST）
- RK3566 本地人体检测（YOLOv3 / v5 / v8 后处理）
- 可选的人体跟踪模块
- Ref 帧差异分析与过滤
- 事件分析模块接口调用
- 配置管理（YAML）
- 日志系统

### 本期不实现

- 多路相机调度
- 前端可视化页面
- 告警平台对接
- 多路扩展预留设计

                           │

### 流程详述
1. 连接相机 RTSP 视频流。
2. 调用查询接口，判断相机是否支持返回人体目标检测结果。
3. **如果相机支持检测结果：**
- 通过 ONVIF 或 REST 接口周期性读取相机检测结果；
- 若检测结果中存在人体目标，则解码对应视频帧；
- 若无目标，则跳过本轮分析。
4. **如果相机不支持检测结果：**
- 按配置帧率解码视频流，获取图像帧；
- 将图像帧送入 RK3566 本地检测算法；
- 若配置启用跟踪模块，则检测结果经跟踪模块输出；
- 若未检测到目标，则跳过本轮分析；
- 若检测到目标，则进入后续分析。
5. 将当前帧与参考帧 Ref 做差异分析：
- 若两帧几乎一致（相似度高于阈值），则跳过后续分析；
- 若存在显著差异，则将当前图像帧或当前帧之后一段时间的短视频送入事件分析模块。
6. 更新 Ref 参考帧。
7. 持续循环执行。

## 4. 术语

|   术语    | 定义 |
|  :---:   |:---:|
|**Ref 帧**|最近一次送入事件分析模块的图像帧，或视频段的最后一帧|
|**相同帧**|当前帧与 Ref 帧的相似度高于设定阈值，视为内容未发生变化|
|**有目标**|检测结果中存在至少一个人体目标|
|**检测结果来源**|相机侧（ONVIF / REST）或本地检测（RK3566 NPU 推理）|

## 5. 相机检测结果接口
**获取方式**
通过配置文件指定获取方式，当前支持：
* ONVIF
* REST
默认不进行鉴权。

**返回数据结构**
```cpp
struct CameraDetectionResult {
    int obj_num;                        // 目标数量
    std::vector<BoundingBox> objs;      // 目标坐标框列表
    int64_t time_stamp;                 // 图像帧时间戳
    int frame_id;                       // 图像帧号
};
```

**同步规则**
* 相机返回的检测结果通过 frame_id 或 time_stamp 与当前视频帧进行匹配同步。
* 若匹配失败或超时，视为本轮无有效检测结果。
* 匹配超时后直接失败，不进行重试，记录日志后继续处理下一帧。

## 6. 跟踪模块
**启用方式**
* 通过配置文件中的 enable_tracker: true/false 控制是否启用跟踪模块。
* **启用时**：检测结果经跟踪模块处理后输出，输出结果包含稳定的 track_id。
* **未启用时**：直接使用检测模块的原始输出。

**跟踪模块职责**
* 为连续帧中的同一人体分配稳定的 track_id
* 输出每个目标的跟踪状态（tentative / confirmed / lost / removed）
* 仅 confirmed 状态的目标进入后续 Ref 帧比较与事件分析流程


## 7. 非功能要求
**开发语言与平台**
* 整个项目基于 C++ 开发
* 目标平台为 RK3566 + Android
* 视频解码采用 FFmpeg + OpenCV
* 模型推理基于 RKNN Runtime（需查询 RK3566 NPU 模型加载 / 推理接口）

**稳定性**
* 支持 RTSP 断流自动重连
* 单路处理支持长时间稳定运行（7×24 小时）

**模块化**
* 处理流程模块化，便于替换检测器
* 检测器后处理支持不同版本 YOLO 解码（YOLOv3 / v5 / v8）

**可观测性**
* 检测链路关键节点需输出日志
* 日志内容至少包括：帧号 / 时间戳、检测目标数量、目标框位置及置信度、各模块耗时

**日志规范**
| 项目	 | 要求 |
| :----: | :----|
|日志级别|	DEBUG、INFO、WARNING、ERROR|
|关键日志级别	| INFO |
| 日志内容 |	必须包含代码位置（文件名 + 行号）、时间戳|
| 输出方式	| 仅输出到日志文件，不输出到控制台 |
| 文件策略 | 单文件存储，不限制文件大小，不做日志轮转 |
| 文件大小 | 不做要求 |

**配置管理**
* 所有配置项统一采用 YAML 格式
* 配置文件中至少包含以下可配置项：
```yaml
# RTSP 配置
rtsp:
  url: "rtsp://192.168.1.100:554/stream"
  reconnect_interval_ms: 3000

# 相机检测结果获取方式
camera_detection:
  enabled: true
  protocol: "REST"             # REST 或 ONVIF
  endpoint: "http://192.168.1.100/api/detection"
  poll_interval_ms: 100

# 本地检测配置
local_detection:
  model_path: "/data/models/yolov8.rknn"
  model_type: "yolov8"         # yolov3 / yolov5 / yolov8
  conf_threshold: 0.5
  detect_interval: 3           # 每 N 帧检测一次
  timeout_ms: 500

# 跟踪模块配置
tracker:
  enabled: true
  confirm_frames: 3
  max_lost_frames: 30

# Ref 帧差异分析配置
ref_frame:
  similarity_threshold: 0.95
  compare_method: "ssim"       # ssim / phash / mse

# 事件分析配置
event_analysis:
  mode: "image"                # image 或 video
  video_duration_sec: 5        # 当 mode=video 时，截取后续视频时长
  video_frame_skip: 3          # 每3帧保留一帧

# 日志配置
logging:
  level: "INFO"
  file_path: "/data/logs/pipeline.log"
```

## 8. 验收标准
**场景 1：相机支持检测结果**
```less
Given  相机检测接口可正常返回人体检测结果
When   检测结果中存在人体目标
Then   系统应解码对应帧并进入 Ref 帧差异分析流程
```

**场景 2：相机不支持检测结果**
```less
Given  相机检测接口不可用或返回不支持
When   视频帧到达
Then   系统应执行本地人体检测
```

**场景 3：跟踪模块启用**
```less
Given  配置文件中 tracker.enabled = true
When   本地检测到人体目标
Then   检测结果应经过跟踪模块输出
And    输出结果中应包含稳定的 track_id
And    仅 confirmed 状态的目标进入后续分析
```

**场景 4：跟踪模块未启用**
```less
Given  配置文件中 tracker.enabled = false
When   本地检测到人体目标
Then   直接使用检测模块的原始输出进入后续分析
```

**场景 5：重复帧跳过**
```less
Given  当前帧与 Ref 帧相似度高于阈值
When   系统执行 Ref 帧差异分析
Then   跳过事件分析模块，继续处理下一帧
```

**场景 6：新变化触发分析**
```less
Given  当前帧与 Ref 帧相似度低于阈值
When   系统执行 Ref 帧差异分析
Then   将当前帧或后续短视频送入事件分析模块
And    更新 Ref 参考帧
```

**场景 7：无目标时跳过**
```less
Given  当前帧未检测到人体目标
When   检测模块返回空结果
Then   系统跳过 Ref 帧比较和事件分析，继续处理下一帧
```

**场景 8：RTSP 断流重连**
```less
Given  RTSP 视频流发生断连
When   系统检测到连接中断
Then   按配置的重连间隔自动尝试重新连接
And    重连过程记录 WARNING 级别日志
```

**场景 9：日志输出**
```less
Given  系统正常运行
When   检测到人体目标
Then   日志文件中应记录：
       - 帧号或时间戳
       - 检测到的人体数量
       - 每个目标框的位置及置信度
       - 代码位置（文件名 + 行号）
       - 记录时间
```

## 9. 其他说明
* 事件分析模块的的输入格式需要同时支持图像帧和短视频片段
* 事件分析接口不用本期开发，已有现成接口，当前可封装一个event_analysis的空接口，待后面联调进行对接
* 相机G60的ONVIF/REST的接口文档暂时没有，可以采用通用的进行开发适配
* Ref 帧的差异比较可以配置是基于人体ROI区域还是整个图像帧
* 无论是否启用跟踪模块，Ref 帧都只保留一帧最新的
* **丢帧策略直接采用丢弃最旧的的帧**
