# 图像/视频事件分析模块

## 1. 目标
将经过目标检测过滤和差异检测过滤的图像帧/视频片段送入事件分析模块，输出分析结果

## 2. 输入
- frames: vector<cv::Mat> // 当输入图像帧时默认长度为1,视频片段时长度为fps * time
- boxes: vector<BoundingBox> // 图像帧或者视频片段的首帧检测到的目标框

## 3. 输出
- void

## 4. 规则
- 只有在检测到目标框，及boxes非空且与Ref有差异时，才执行事件分析
- 分析失败时，不得阻塞主循环，应记录错误日志，并继续后续帧处理
- 如果需要生成视频进行事件分析，则从检测到有差异的帧开始，按照指定配置的分辨率，配置的时长进行录制然后送入事件分析

## 5. 验收标准
- 符合条件的帧或者视频片段才能进入分析
- 分析异常不导致RTSP主流程推出
- 事件分析只是将图像或者视频通过提供的event_analysis接口送入分析模块，不在进行后续处理
- event_analysis为空接口，后续跟其他模块进行对接
